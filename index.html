Abstract
This report documents the design and implementation of a Classroom/Lab Access Control System developed for EFB 2073 – Microprocessors and Computer Architecture using the NUCLEO-F401RE Cortex-M4 microcontroller and the ARM Mbed development environment. The system integrates a 4×4 matrix keypad for PIN-based authentication, a DHT11 digital sensor for real-time temperature and humidity monitoring, an SSD1306 OLED display for system feedback, and three LED indicators for visual status signalling. Access is granted only when both conditions are satisfied: the correct 4-digit PIN is entered and the room temperature is below 35 °C. Overheating triggers an automatic access denial with a distinct yellow LED warning regardless of PIN correctness. Additional features include a 30-second lockout after three consecutive failed attempts, an admin mode for PIN reconfiguration, occupancy tracking, and a rolling access log storing the last three entry attempts. All firmware was written in C++ using Mbed OS with no external libraries required.
Section 1

Background

1.1 Project Introduction

Physical access control is a fundamental requirement in laboratory and research environments where sensitive equipment, intellectual property, and safety-critical conditions must be protected. Traditional key-based systems offer no audit trail and cannot respond dynamically to environmental conditions such as overheating. Microcontroller-based electronic access control systems address these limitations by combining credential-based authentication with real-time environmental sensing.

This project implements the system on the NUCLEO-F401RE development board equipped with the STM32F401RE Cortex-M4 microprocessor, using ARM Mbed OS as the software development framework. The system controls physical access based on two simultaneous conditions - a correct 4-digit PIN and a safe room temperature below 35 °C.

1.2 Problem Statement

Laboratory environments at universities house expensive equipment and sensitive research materials requiring controlled access. Conventional lock-and-key mechanisms cannot prevent access under unsafe environmental conditions such as electrical overheating, and they provide no access audit trail. There is therefore a need for an intelligent embedded access control system that simultaneously authenticates users and monitors environmental safety, denying entry automatically when conditions are unsafe.

1.3 Project Objectives

  1. 1

    Implement a 4-digit PIN authentication system using a 4×4 matrix keypad with lockout protection after three consecutive failed attempts.

  2. 2

    Integrate a DHT11 sensor for real-time room temperature and humidity monitoring, with automatic access denial when temperature exceeds 35 °C.

  3. 3

    Display live system status, PIN entry feedback, temperature, humidity, and an access log on an SSD1306 OLED display via I2C.

  4. 4

    Implement occupancy tracking that toggles the lab status between VACANT and OCCUPIED upon each successful access grant.

  5. 5

    Develop an admin mode allowing authorised users to change the system PIN via the onboard push button.

  6. 6

    Demonstrate structured embedded firmware development using C++ in ARM Mbed, incorporating GPIO, I2C, timers, and digital sensor interfaces.

Section 3

Firmware / Software Development

3.1 Development Environment

All firmware was developed using the ARM Mbed online compiler at os.mbed.com, targeting the NUCLEO-F401RE. A single main.cpp source file contains all firmware modules. No external libraries were required - the OLED driver, DHT11 driver, and keypad scanner were all written from scratch using Mbed OS built-in APIs. The compiled .bin file was flashed by drag-and-drop onto the NUCLEO USB drive.

3.3 Main Loop Logic

Environment Refresh
Every 3 seconds, dht11_read() updates temperature and humidity. If temperature > 35 °C, yellow LED toggles continuously (~0.17 Hz blink).
Lockout Handling
If locked_out is true, remaining seconds are computed from the hardware timer and displayed on the OLED. Keypad input is skipped entirely.
Admin Mode
If the onboard button reads LOW (with 50 ms debounce), the system prompts for the current PIN before entering change_pin_mode().
Keypad Processing
A = show access log  |  # = confirm PIN  |  * = clear entry  |  0–9 = append digit to PIN buffer (max 4 characters).

3.4 Key Code Snippets

DHT11 Read Function

main.cpp 
bool dht11_read(float &temp, float &hum) {
    uint8_t data[5] = {0};
    dht_pin.output(); dht_pin = 0; thread_sleep_for(18);
    dht_pin = 1; wait_us(30); dht_pin.input();
    // Wait for sensor acknowledgement
    int t = 10000;
    while (dht_pin == 1 && t--); if (t <= 0) return false;
    t = 10000;
    while (dht_pin == 0 && t--); if (t <= 0) return false;
    // Receive 40 bits
    for (int i = 0; i < 40; i++) {
        t = 10000; while (dht_pin == 0 && t--);
        wait_us(35);
        if (dht_pin == 1) data[i/8] |= (1 << (7 - (i%8)));
        t = 10000; while (dht_pin == 1 && t--);
    }
    // Checksum validation
    if (data[4] != ((data[0]+data[1]+data[2]+data[3]) & 0xFF))
        return false;
    hum  = data[0] + data[1] * 0.1f;
    temp = data[2] + data[3] * 0.1f;
    return true;
}

Keypad Scanning

main.cpp 
char keypad_read() {
    DigitalOut *rows[4] = {&r1, &r2, &r3, &r4};
    DigitalIn  *cols[4] = {&c1, &c2, &c3, &c4};
    for (int r = 0; r < 4; r++) {
        r1=0; r2=0; r3=0; r4=0;
        *rows[r] = 1;
        wait_us(100);
        for (int c = 0; c < 4; c++) {
            if (*cols[c] == 1) {
                while (*cols[c] == 1); // wait for release
                return keymap[r][c];
            }
        }
    }
    return 0;
}

Access Grant & Deny Logic

main.cpp 
void grant_access() {
    occupied = !occupied;
    log_attempt(1);            // 1 = GRANTED
    led_green = 1; led_red = 0; led_yellow = 0;
    oled_clear();
    oled_print(0, occupied ? "LAB OCCUPIED" : "LAB VACANT");
    oled_print(1, "ACCESS GRANTED");
    thread_sleep_for(3000);
    led_green = 0; wrong_attempts = 0;
    reset_entry(); show_main_screen();
}

void deny_access(bool overheat) {
    log_attempt(overheat ? 3 : 2);  // 2=DENIED 3=OVERHEAT
    led_red = 1; led_green = 0; wrong_attempts++;
    if (overheat) {
        oled_print(1, "ACCESS DENIED");
        oled_print(2, "OVERHEAT RISK");
        led_yellow = 1;
    } else if (wrong_attempts >= 3) {
        oled_print(1, "LOCKED 30S");
        locked_out = true;
        lockout_timer.reset(); lockout_timer.start();
    }
}

Section 4

Project Outcomes & Accomplishments

4.1 Testing Procedure

Hardware testing was conducted by systematically exercising each functional scenario on the physical NUCLEO-F401RE prototype. Each test case was executed a minimum of three times to confirm consistent behaviour. The DHT11 threshold test was performed by warming the sensor with a heat source to raise the temperature above 35 °C. PIN lockout was triggered by deliberately entering three consecutive wrong PINs and observing the countdown. Admin mode was verified by holding the button, entering the current PIN, setting a new PIN, and confirming the new PIN grants access while the old one does not.

4.3 Observed System Behaviour

The SSD1306 OLED display correctly rendered all system states across its four rows. Row 0 permanently showed occupancy status. Row 1 updated contextually to show PIN prompts, access results, or lockout countdown. Rows 2 and 3 refreshed live temperature and humidity every 3 seconds. The DHT11 sensor produced stable readings throughout; the built-in checksum validation discarded corrupted reads without interrupting the display. The keypad scanning algorithm correctly identified all 16 keys without false triggers, with the 100 µs settle delay per row sufficient to eliminate contact bounce.

4.5 Limitations & Possible Improvements

  • DHT11 accuracy is ±2 °C and ±5% RH. Replacing with a DHT22 or SHT31 would improve precision for safety-critical thresholds.
  • The access log is stored in volatile RAM and is lost on power-off. Persisting to EEPROM or flash would allow historical records to be retained.
  • The OLED font covers only uppercase letters and digits. Extending to lowercase and special characters would improve readability.
  • The lockout timer uses a polling approach in the main loop. A hardware timer interrupt would provide a more precise and CPU-independent countdown.
  • Only a single shared PIN is supported. A multi-user system with individual PINs and identifiers would significantly increase real-world utility.
Conclusion

Conclusion

This project successfully demonstrates the design and implementation of a complete embedded access control system on the NUCLEO-F401RE microcontroller using the ARM Mbed development environment. The system integrates a 4×4 matrix keypad, DHT11 environmental sensor, SSD1306 OLED display, three LED indicators, and an onboard push button to deliver a functional and multi-featured access control solution.

All six project objectives were met. PIN-based authentication with lockout protection, environment-conditional access denial, real-time temperature and humidity display, occupancy tracking, access logging, and admin PIN reconfiguration were all implemented and verified in hardware. The use of Mbed OS allowed rapid development of complex multi-peripheral firmware while maintaining structured, readable, and modular code. The project demonstrates how microcontroller-based embedded systems can solve real-world engineering problems in security and environmental monitoring.

// ... font tables, OLED, DHT11, keypad, access logic ...
// See Section 3.4 for key function implementations.

int main() {
    i2c.frequency(100000); oled_init();
    led_green = 0; led_red = 0; led_yellow = 0;
    dht11_read(temperature, humidity); show_main_screen();
    env_timer.start(); lockout_timer.reset();

    while (true) {
        if (env_timer.elapsed_time() >= 3s) {
            env_timer.reset(); dht11_read(temperature, humidity);
            led_yellow = (temperature > 35.0f) ? !led_yellow : 0;
            if (!locked_out) show_main_screen();
        }
        if (locked_out) { /* countdown logic */ continue; }
        if (btn == 0)    { /* admin flow */ }
        char key = keypad_read(); if (!key) continue;
        if      (key == 'A') show_access_log();
        else if (key == '#') { /* PIN confirm */ }
        else if (key == '*') { reset_entry(); show_main_screen(); }
        else if (key >= '0' && key <= '9')
            if (pin_index < 4) { entered_pin[pin_index++] = key; show_pin_dots(); }
    }
}
